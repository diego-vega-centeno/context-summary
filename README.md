# Context Summary

**Context Summary** is an AI-powered dashboard that ingests GitHub Pull Request data and reconstructs the development narrative. Instead of just showing the current status, it uses Gemini to answer *"how did we get here?"* by recovering the story, key decisions made, blocking points, and recommended next steps.

For full product specs and requirements, refer to the [project-spec.md](https://github.com/diego-vega-centeno/context-summary/blob/main/project-spec.md).

---

## Features

- **PR Tracking**: Track public PRs by URL without requiring repository ownership. Detects stale PRs (no activity in 7+ days).
- **Dashboard View**: View aggregate stats (total, open, stale, merged, closed) and a visual kanban board mapping PR statuses.
- **PR Story View**: Chronological AI-generated narratives, categorized into collapsible sections:
  - What was built and why.
  - Key decisions (who made them, when, and context).
  - Blocking points (context and who we are waiting on).
  - Suggested next steps.
- **Manual & Automated Sync**: Manual refresh button for on-demand sync.

---

## Tech Stack

- **Frontend & Backend**: Next.js 16 (App Router)
- **Database**: PostgreSQL (Supabase)
- **AI Model**: Gemini 2.5 Flash Lite (via `@google/genai`)
- **Authentication**: NextAuth.js v5 (Supporting Credentials & Google OAuth)
- **Styling & Icons**: Tailwind CSS v4 & Lucide Icons

---

## Database Schema

The database tables are defined in [schema.sql](file:///C:/Users/gonta/D/study/full%20stack/projects/context-summary/schema.sql):

- **`users`**: Managed via credentials/oauth. Stores user settings such as custom stale threshold (`stale_days`).
- **`tracked_work_items`**: Holds repository information owner, name, id, title, status, and activity tracking timestamps.
- **`work_item_summaries`**: Stores the structured JSON representation of the narrative summary generated by Gemini.
